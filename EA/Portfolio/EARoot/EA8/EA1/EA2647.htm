Starting with a DBF package, each class owner builds his methods for the feature. He extends his class-based test cases and performs class-level (unit) testing. The feature team inspects the code, perhaps before unit test, as determined by the chief programmer. Once the code is successfully implemented and inspected, the class owner checks in his class(es) to the configuration management system.
