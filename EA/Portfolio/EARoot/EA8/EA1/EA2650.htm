The team identifies the features, groups them hierarchically, prioritizes them, and weights them.<br/>In subsequent iterations of this process, smaller teams tackle specialized feature areas. Domain members participate in many yet not all of those follow-up sessions.<br/><br/><b>Entry Criteria</b><br/>The modeling team has successfully completed FDD Process #1, Develop an Overall Model.<br/><br/><br/>From the Overall Design Model the Design Team identifies Feature Sets and Features, putting them into the feature list, grouping them hierarchically and providing a rough level of effort estimate and dependencies for each feature. The result should be a prioritized Feature List. Features should be kept small enough that they can be completed within a single week. An initial level of effort estimate should be provided for each feature. The target size for each feature should be that it can be implemented within a single 40 hour period. Keeping features at roughly the same level of effort will improve the reliability of estimating release dates and simplify the capacity planning and efficiency process.<br/><br/>
