A chief programmer takes the next feature, identifies the classes likely to be involved, and contacts the corresponding class owners. This feature team works out a detailed sequence diagram. The class owners write class and method prologs. The team conducts a design inspection.<br/><br/><b>Entry Criteria</b><br/>The planning team has successfully completed FDD Process #3, Plan by Feature.<br/><br/>Start of the “development” cycle. For each feature in the planning list, the Lead Developer creates a Feature Team by assembling the developers that are involved in the planned features. The Feature Team will analyze and walk through requirements, building a class model and sequence diagrams as required.<br/>Each class owner/lead developer in the feature team implements the methods and classes defined in the Design By Feature phase. As each class and method is implemented, unit testing is continuously performed. The Feature Team then integrates the classes that make up the feature, performing “end-to-end” testing of the feature and conducts a code review of the features. As features pass testing, they are checked in and designated as “Promoted to Build” (Deployed).<br/>
